技术规格与参数

PCT-100 详细硬件参数、电气特性及软件架构说明

1. 主控芯片参数

型号ESP32-C3 (ESP32-C3FH4)
核心架构RISC-V 32-bit 单核
主频最高 160 MHz
SRAM400 KB
Flash4 MB (Quad SPI)
WiFi802.11 b/g/n (2.4 GHz), HT20/HT40
蓝牙Bluetooth 5.0 LE
ADC2 × 12-bit SAR ADC, 最高 100ksps
GPIO22 个可编程 IO
工作电压3.0V ~ 3.6V (板载 3.3V LDO)

2. 传感器规格

DS18B20 数字温度传感器

接口1-Wire（单总线）
测量范围-55°C ~ +125°C
精度±0.5°C（-10°C ~ +85°C）
分辨率9 ~ 12 bit（可配置，默认 12 bit）
转换时间750ms (12 bit 模式)
供电3.0V ~ 5.5V（寄生/外部供电）

LDR 光敏电阻

接口模拟电压输出 (ADC 采样)
采样电路10kΩ 上拉分压
ADC 分辨率12 bit (0 ~ 4095)
响应时间约 20ms

3. 电气特性

供电电压5V DC（USB Type-C 或接线端子）
最大功耗约 500mA (含继电器驱动)
继电器触点额定250V AC / 10A (阻性负载)
继电器驱动方式有源高电平 (GPIO 6 / 7)
OLED 供电3.3V (板载 LDO)
WS2812 供电3.3V (板载 LDO)
按键电平低电平有效 (内部上拉)
供电建议：当两路继电器同时驱动大负载时，建议使用 5V / 2A 以上电源适配器供电，避免 USB 端口供电不足导致系统不稳定。

4. 通信协议栈

MQTT 数据格式

状态上报 JSON 示例：

{
  "power": true,
  "mode": "auto",
  "temp": 26.5,
  "lux": 450,
  "led_relay": true,
  "fan_relay": false
}

5. 软件架构

基于 Arduino 框架 + ESP32 Arduino Core，软件分为四层：

ArduinoJson PubSubClient U8g2 (OLED) Adafruit NeoPixel DS18B20 by matmunk FreeRTOS Preferences (NVS)

文件结构

层级模块文件职责
编排层src.ino, scheduler, device.h初始化顺序、任务注册、loop 轮询
业务层controller, mqtt_handler, ws2812_indicator控制策略、阈值判断、状态通知
传输层wifi_drv, mqtt_client_drv, serial_cmd网络通信、串口命令解析
驱动层key, relay, ldr, ds18b20_drv, oled_ui, ws2812_drv硬件寄存器操作、传感器协议

6. 时序参数

按键去抖时间20ms（状态机采样周期）
光照采样周期50ms
温度采样周期2000ms（DS18B20 转换时间 750ms）
WS2812 刷新率50 fps（20ms 周期）
OLED 刷新周期3000ms（事件驱动可立即刷新）
MQTT 重连间隔5000ms（带指数退避）
WiFi 重连间隔渐进式，最小 1000ms
串口波特率115200 baud

7. 环境要求

工作温度-10°C ~ +60°C
存储温度-20°C ~ +85°C
工作湿度10% ~ 90% RH（无凝露）
防护等级IP20（室内使用）
安装方式导轨安装 / 桌面放置
注意：本装置为实训教学设备，请在教师指导下操作。继电器输出端涉及 220V 高压，接线前务必确认断电。
