应用场景与操作指南

从实训教学到物联网开发，PCT-100 的多场景应用与快速上手指南

1. 典型应用场景

🧪

化工实训教学

作为化工自动化控制实训平台，学生可通过本装置学习传感器采集、继电器控制、阈值设定等工业控制基础知识。配合 MQTT 物联网通信，理解工业 IoT 架构。

🏭

环境监控系统原型

适用于环境监测系统原型开发：温度越限告警、自动通风控制、光照自动补偿。可作为智能农业、仓储管理等场景的技术验证平台。

🔌

物联网通信实验

基于 MQTT 协议的端云通信实验平台。支持串口命令行配置 MQTT 参数，体验完整的物联网数据采集 → 上传 → 远程控制闭环流程。

💻

嵌入式软件开发实训

四层架构的嵌入式 C 项目范本，涵盖驱动开发、通信协议实现、业务逻辑编排。适合嵌入式软件工程实训课程。

🎓

毕业设计 / 课程项目

提供完整的硬件原理和软件源码，可在此基础上进行功能扩展：如增加更多传感器、接入云平台、数据可视化仪表板开发等。

⚙️

自动化控制演示

演示自动控制逻辑——光照低于阈值自动开灯、温度超限自动启动风扇。手动模式可独立控制每路继电器，展示工业控制基本逻辑。

2. 实训操作流程

一次完整的实训流程通常包含以下步骤：

  1. 上电准备 — 连接 5V 电源，串口线接至 PC，打开串口终端（115200 baud）。等待系统初始化完成，OLED 显示初始状态。
  2. WiFi 配网 — 使用 wifi scan 扫描可用网络，wifi conn n password 连接。连接成功后指示灯从红绿交替变为其他状态。
  3. 配置 MQTT — 通过 mqtt set ip=服务器IP、mqtt set user=用户名、mqtt set pass=密码 配置 MQTT 参数，配置自动持久化至 NVS。
  4. 闭合总闸 — 按下 KEY1 闭合总闸，装置进入默认自动模式。此时 LDR 检测光照、DS18B20 检测温度，系统根据阈值自动控制继电器。
  5. 模式切换 — 短按 KEY2 在手动模式中循环继电器状态（00→01→10→11），长按 KEY2 切换自动/手动模式。
  6. 远程监控 — 使用 MQTT 客户端订阅状态主题，实时查看设备数据；发送控制命令远程操作继电器。

3. 快速上手指南

首次使用配置流程

硬件连接检查清单

  • DS18B20 传感器连接至 GPIO 10（注意数据线不要接反）
  • LDR 光敏电阻连接至 GPIO 1（模拟输入）
  • OLED 显示屏 I2C 连接至 GPIO 4 (SDA) / GPIO 5 (SCL)
  • WS2812 RGB LED 连接至 GPIO 0
  • 继电器模块信号线连接至 GPIO 6 (LED) / GPIO 7 (FAN)
  • 按键连接至 GPIO 20 (KEY1) / GPIO 21 (KEY2)

4. 串口命令速查

命令说明
help显示全部命令列表
get status查看当前全部系统参数
set temp=30.0设置温度阈值（°C）
set light=300设置光照阈值（ADC 值）
set mode=auto切换为自动模式
set mode=manual切换为手动模式
mqtt status查看 MQTT 连接状态与当前配置
mqtt reconnect强制重新连接 MQTT
wifi status查看 WiFi 连接信息
wifi forget清除保存的 WiFi 凭据

5. MQTT 远程控制

命令格式

向 chemctrl/{device_id}/command 发布 JSON 指令：

命令JSON 示例说明
set_relay {"cmd":"set_relay","id":3,"on":true} id=3 为 LED，id=4 为 FAN
set_mode {"cmd":"set_mode","mode":"auto"} auto 或 manual
set_threshold {"cmd":"set_threshold","type":"temp","val":32.0} type: temp 或 light
get_status {"cmd":"get_status"} 触发设备上报完整状态
reboot {"cmd":"reboot"} 远程重启设备

状态上报示例

设备在状态变化时自动发布到 chemctrl/{device_id}/status：

{
  "power": true,
  "mode": "auto",
  "temp_val": 26.5,
  "temp_th": 30.0,
  "lux_val": 450,
  "lux_th": 300,
  "led_relay": true,
  "fan_relay": false
}

6. 二次开发指南

开发环境搭建

  1. 安装 Arduino CLI — 从 arduino.cc 下载安装或使用包管理器安装 arduino-cli。
  2. 安装 ESP32 核心 — 运行 arduino-cli core install esp32:esp32 安装 ESP32-C3 支持。
  3. 安装所需库 — 通过 Library Manager 安装 ArduinoJson、U8g2、PubSubClient、Adafruit NeoPixel、DS18B20 by matmunk。
  4. 编译烧录 — 使用 arduino-cli compile --fqbn esp32:esp32:esp32c3 --upload -p /dev/ttyACM0 src/src.ino 编译并上传。
  5. 架构原则 — 保持四层单向依赖：驱动不引用业务、业务不引用编排。新增模块遵循 xxx_init / xxx_loop 模式。
